Наблюдатель/Observer

Наблюдатель это поведенческий паттерн, который позволяет одним объектам (Наблюдателям) реагировать на изменения происходящие в других объектах (Издателях)

Псевдокод класса Наблюдателя

interface IObserver {
    update(param);
}

class Observer implements IObserver {
    update(param) {
        //Логика реагирования на возникшее событие
    }
    //Логика необходимая классу по его назначению
}
Псевдокод класса Издателя

interface ISubject {
    subscribe(observer: IObserver);
    unsubscribe(observer: IObserver);
    notify();
}

class Subject implements ISubject {
    private observers = [];
    private parameter = '';

    subscribe(observer: IObserver) {
        observers.add(observer);
    }
    unsubscribe(observer: IObserver) {
        observers.remove(observer);
    }
    notify() {
        observers.forEach((observer) => observer.update(parameter));
    }
    //Логика необходимая классу по его назначению
    sendEvent() {
        this.notify();
    }
}
Псевдокод использования класса

const subject = new Subject();

const someObserver = new Observer();
const anotherObserver = new Observer();

subject.subscribe(someObserver);
subject.subscribe(anotherObserver);

subject.senEvent();
subject.notify();
Одиночка/Singleton

Одиночка это порождающий паттерн, который обеспечит существование только одного экземпляра класса

Псевдокод класса

class Singleton {
    private static instance = new Singleton();
    private constructor() {}

    static getInstance() {
        return Singleton.instance;
    }
}
Псевдокод использования класса

const first = Singleton.getInstance();
const second = Singleton.getInstance();
first === second //true

Состояние/State

Состояние это поведенческий паттерн проектирования, который позволяет объектам менять свое поведение при изменении внутреннего состояния

Характерные черты:
  • Каждое состояние знает о необходимых смежных состояниях и переключается на них при определенных условиях
  • Последовательность состояний непрерывна и предопределена
  • В каждом состоянии приложение или объект показывает разную реакцию на однотипные действия

Стратегия/Strategy

Стратегия это поведенческий паттерн, который определяет набор схожих алгоритмов и позволяет их подменять во время выполнения

Характерные черты:
  • Каждая стратегия является алгоритмом, который независит от других стратегий и выбор стратегии выполняет объект использующий стратегии
  • Каждая стратегия в приложении или объекте показывает разную реакцию на однотипные действия

Фабрика/Factory

Фабрика это порождающий паттерн, который определяет общий интерфейс для создания объектов, позволяя изменять тип создаваемых объектов

Характерные черты:
  • Каждая фабрика порождает объекты со схожей функциональностью, но различной реализацией
