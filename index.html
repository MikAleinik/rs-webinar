Кто такой фронтендер

В общих фразах...
  • Создает внешнюю часть сайта или веб-приложения, верстает, добавляет текст, изображения, кнопки и т.д.
  • Добавляет страницам интерактивность и отзывчивость
  • Тестирует проект на всех этапах разработки, ищет и устраняет баги
  • Делает так, чтобы веб-страницы корректно открывались на разных устройствах и браузерах
По мнению HR...
  • Верстать веб-сайты использованием HTML5, CSS3 и Javascript
  • Владеть методологией верстки
  • Правка верстки на существующих сайтах
  • Адаптация верстки под мобильные устройства
  • Опыт работы с системами контроля версий Git, GitHub, GitLab, BitBucket
  • Понимание работы HTTP/HTTPS, WS протоколов
  • Опыт работы с различными API, REST API
  • React, TypeScript, Redux
  • Angular, TypeScript, NgRx, RxJs
  • Vue, Vuex
  • Знание и опыт работы с Vite и Webpack для сборки проектов
  • Библиотеки компонентов
  • Понимание архитектурных принципов и паттернов
  • SOLID, DRY, KISS, ООП
  • Написание чистого, хорошо структурированного, кросс-браузерного, тестируемого и масштабируемого кода

Базовые технические навыки

Окружение приложения

Разрабатывая приложение необходимо понимать то окружение, для которого оно предназначено (в котором будет выполняться), знать его особенности и механизмы

Для начинающего фронтед разработчика это окружение, чаще всего, это сеть Internet.

На начальном этапе нужно понимать

  • основы работы сети Internet
  • доменное имя, хостинг
  • протокол соединения
  • как работает браузер
  • что значит клиент, сервер
  • ...
Три кита frontend разработчика
  • HTML
  • CSS
  • JavaScript
HTML

Определяет что будет на вашей странице и в какой последовательности оно будет отображаться

  • Структура html документа
  • Виды тегов и их особенности
  • Семантика в верстке
  • SEO
CSS

Определяет как будет выглядеть ваша страница. Может изменить последовательность отображения

  • Таблицы стилей
  • Позиционирование
  • Адаптивная и резиновая верстка
  • Кроссбраузерность
  • Семантика в стилях
JavaScript

Определяет интерактивность вашей страницы. Может изменить и HTML, и CSS

  • Базовые конструкции языка
  • Структуры данных
  • Функции
  • Контекст исполнения
  • Асинхронность
  • API браузера
И не только для frontend
  • Поиск информации и документации
  • Чтение информации и документации
  • Понимать чужой код
  • Понимать свой код
  • Воспринимать критику и ревью кода

Базовые инструменты

Стилистика кода
  • ESLint
  • Prettier

Позволяют сделать ваш код единобразным, убрать мелкие "огрехи" и, в зависимости от настроек, применить к нему правила лучших практик в организации стиля кода

Системы контроля версий
  • Git (распределенные)
  • Subversion (локальные)

Системы контроля версий отслеживают все вносимые в код изменения в специальной базе данных. При обнаружении ошибки разработчики могут вернуться назад и выполнить сравнение с более ранними версиями кода для исправления ошибок, сводя к минимуму проблемы для всех участников команды

Менеджеры пакетов
  • npm
  • pnpm
  • yarn

Позволяют использовать код написанный и опубликованный другими разработчиками и использовать его как часть приложения, для целей оптимизации кода, отладки, форматирования, сборки

Сборщики модулей
  • Vite
  • Webpack
  • esbuild
  • Rollup
  • Parcel

Выполняют сборку разрозненных JS модудей приложения в единый файл (или в несколько файлов), который в дальнейшем подключается при загрузке страницы.

Имеют множество настроек упрощающих работу не только с JS модулями, но и со стилями, картинками, шрифтами

Продвинутые инструменты

Фреймворки и библиотеки JS
  • React
  • Vue
  • Angular

Определяют единообразную структуру приложения или его части, предоставляет механизмы и/или инструменты для решения однотипных задач, уменьшает время разработки приложения

Библиотеки UI
  • Tailwind CSS
  • Material UI
  • Ant Design
  • Radix UI

Стандартизирут элементы пользовательского интерфейса, зачастую имеют готовые решения по элементам выходящим за рамки типовых элементов HTML разметки

Препроцессоры
  • SaaS
  • PostCSS
  • Less
  • Stylus

Это приложение, которое определяет свой собственный синтаксис, расширяет возможности чистого CSS, добавляя такие опции как: примеси, вложенные правила, селекторы наследования и может сгенерировать из него CSS код
