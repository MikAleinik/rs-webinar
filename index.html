Что такое SPA приложение.
  • Это реализация веб-приложения, которое загружает только один веб-документ, а затем обновляет содержимое тела этого единственного документа с помощью API-интерфейсов JavaScript, таких как и Fetch, XMLHttpRequest когда нужно показать другой контент. (MDN)

  • Одностраничное (SPA).
  • Прогрессивно улучшенное одностраничное (PESPA).

Признаки SPA приложения.
  • Маршрутизация между компонентами по строке запроса браузера.
  • Генерирует собственные элементы без участия сервера.
  • Хранит собственное состояние при переходах.
Особенности SPA приложения.
  • Трудности индексации страниц(ы) поисковыми системами.
  • Более сложная, при первоначальном знакомстве, реализация.
  • Требуется переадресация запросов на стороне сервера на единую точку входа.
Преимущества и недостатки SPA приложения.

Каждая архитектура имеет свои сильные и слабые стороны. Рассмотрим плюсы и минусы SPA в сравнении с PEMPA приложением.

Действия браузера при загрузке страницы:

  • Отправляет запрос на получение исходного html документа.
  • Последовательный разбор (парсинг) полученного html документа.
  • Загрузка файлов стилей.
  • Загрузка файлов скриптов.
  • Создание DOM.
  • Создание CSSOM.
  • Рендер, компоновка и отрисовка.
Преимущества SPA приложения.
  • Хранение состояния - улучшает отзывчивость приложения в тех частях, которые используют ранее загруженные/введенные данные.
  • Разделение клиент/сервер - на сервере не остается логики необходимой для отрисовки клиента.
  • Создание DOM - не требуется выполнять полностью при переходах между страницами, выполняется модификация существующей модели. Так же нет необходимости в повторной загрузке, парсинге и создании ранее использованных элементов/компонентов.
  • Создание CSSOM - не требуется выполнять полностью при переходах между страницами, при необходимости дополняется существующая модель.
Недостатки SPA приложения.
  • Хранение состояния приложения - ответственность за актуальность состояния (данных) требует уделять этому больше внимания.
  • Размер приложения - из-за необходимости выполнения дополнительной логики, генерации компонентов, маршрутизации, хранения состояния появляется дополнительный код.
  • Производительность - большее количество кода может приводить к ее снижению на слабых устройствах.
  • Каскад зависимостей - приложение разбито на отдельные компоненты, что может приводить к необходимости ожидания загрузки компонентов перед загрузкой данных. Например: app.js -> component.js -> response.json -> image.webp
